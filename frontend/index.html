circuits/AgeVerify.circom 
// ZK Circuit: Prove age >= 18 without revealing age
pragma circom 2.0.0;
include "circomlib/circuits/comparators.circom";

template AgeVerify() {
  // Private: never sent to verifier
  signal private input age;
  // Public: verifier sees this
  signal input minAge;
  signal output isAdult;

  // Check: age >= minAge
  component ageCheck = GreaterEqThan(8);
  ageCheck.in[0] <== age;
  ageCheck.in[1] <== minAge;
  isAdult <== ageCheck.out;

  // CRITICAL: Proof is INVALID unless isAdult == 1
  isAdult === 1;
}

component main {public [minAge]} = AgeVerify();

How ZK-SNARKs Work Here

What is ZK?

Zero-Knowledge proofs let you prove a statement is true without revealing WHY it's true. Like proving you know a password without saying the password.

The Constraint

The circuit has one key rule: isAdult === 1. If age < 18, this fails and NO valid proof can be generated — mathematically impossible.

The Proof

Groth16 outputs 3 elliptic curve points (π_a, π_b, π_c). These encode your computation cryptographically — revealing nothing about the private input.

Verification

The verifier runs one pairing equation: e(π_a, π_b) = e(α,β) · e(pubInputs,γ) · e(π_c,δ). Passes → age ≥ 18. Fails → reject.
